# Flutterスマホゲーム企画書: プロ野球フロントライン

## 1. 企画サマリー

- 仮タイトル: プロ野球フロントライン
- ジャンル: 実在プロ野球チームを題材にした球団経営・監督シミュレーション
- 対応: iOS / Android
- 想定開発: Flutter + サーバーAPI + ライブ運用
- 基本プレイ: 無料 + シーズンパス + 任意課金
- コア体験: 「実在チームのGM兼監督として、現実のシーズンと連動しながら采配・育成・補強で1日1試合を戦う」
- 初期重点球団: 中日ドラゴンズを必須対応。最初の検証体験は「ドラゴンズを自分の采配で勝たせる」に寄せる

プロスピAのようなリアルアクション・選手収集の強豪と正面衝突せず、野球好きが日常的に話している「今日の打順ならどうする」「この二軍選手を上げたい」「トレードするなら誰か」をゲーム化する。

## 2. 企画意図

日本の野球スマホゲーム市場は、リアル3Dアクションとキャラクター育成が強い。一方で、現実のプロ野球ファンが毎日楽しめる「監督・編成・育成・データ分析」寄りのスマホ体験はまだ薄い。

本企画は、実在チーム・実在選手への愛着を入口にしつつ、操作のうまさではなく意思決定のうまさで勝つゲームにする。プレイ時間は1セッション3から7分を基本にし、通勤中・試合前・試合後に触れる設計とする。

特に中日ドラゴンズは初期検証の中心に置く。ドラゴンズファンは投手運用、守備、若手起用、得点力改善、バンテリンドームの球場特性など、監督・編成シミュレーションに落とし込みやすい論点が多い。最初から全12球団を同じ密度で作るより、ドラゴンズで「分かる、悩む、語りたくなる」体験を深く作り、その後に他球団へ横展開する。

## 3. 競合分析

### 3.1 国内主要タイトル

| タイトル | 強み | 弱み / 空白 |
| --- | --- | --- |
| プロ野球スピリッツA | 実在NPB選手、リアル3D、選手収集、対人戦、長期運用力 | アクション操作とガチャ収集が中心。球団経営や現実シーズン連動の意思決定は主役ではない |
| 実況パワフルプロ野球 / 栄冠系 | 育成ドラマ、キャラ性、短期目標の分かりやすさ | 実在プロ球団のフロント体験からは距離がある |
| プロ野球ペナント系シミュレータ | ドラフト、成績、ペナント進行などデータ好きに刺さる | UI/演出/ライブ運用が弱く、広いスマホユーザーに届きにくい |
| 既存カジュアル野球アプリ | 短時間で遊べる、導入が軽い | 実在プロ野球ファンの深い語り・継続欲求を拾いにくい |

### 3.2 海外主要タイトル

| タイトル | 強み | 弱み / 空白 |
| --- | --- | --- |
| MLB 9 Innings / MLB Rivals | MLB公式ライセンス、リアル選手、育成・対戦・収集 | 日本のNPBファンの日常文脈には直結しにくい |
| OOTP Go | 本格GMシミュレーション、データ量、コアファン向けの深さ | スマホでの軽快さ、初心者導線、日次ライブ感は重くなりやすい |
| Baseball Clash等のカジュアル系 | 分かりやすい対戦、短時間プレイ | プロ野球の実在チーム愛やシーズン追体験とは別物 |

### 3.3 市場の空白

1. 実在プロ野球チームを「操作する」のではなく「任される」体験
2. 現実の試合日程・ニュース・選手状態をゲーム内意思決定に変換する体験
3. アクションが苦手でも、野球知識と采配で勝てるスマホゲーム
4. データ好きには深く、ライト層にはおすすめ采配で遊べる二層構造
5. ガチャ偏重ではなく、応援・分析・予想・育成に課金価値を置く設計

## 4. 提案コンセプト

### 4.1 ワンフレーズ

「あなたの推し球団を、今日からあなたが勝たせる。」

### 4.2 コアターゲット

- 20代から50代のプロ野球ファン
- 毎日順位表・スタメン・成績をチェックする層
- プロスピAは知っているが、アクション対戦やガチャ疲れがある層
- パワプロの栄冠/ペナント、Football Manager、OOTPのような編成ゲームが好きな層

### 4.3 サブターゲット

- 実況・解説動画、SNSの野球議論が好きなライトファン
- 野球観戦中のセカンドスクリーンとして遊びたいユーザー
- 推し選手の育成や起用にこだわりたいファン

## 5. ゲーム概要

プレイヤーは実在プロ野球チームのGM兼監督となり、以下を毎日行う。

1. チーム状況確認
   - 順位、直近成績、選手コンディション、相手先発、球場特性を確認
2. 編成
   - 一軍/二軍入れ替え、打順、守備位置、先発ローテ、ブルペン役割を決定
3. 試合
   - 基本は高速シミュレーション
   - 勝負所だけ介入し、代打・継投・盗塁・バント・守備固めを選ぶ
4. 試合後
   - 選手評価、ファン支持率、フロント評価、育成ポイント、ニュースが発生
5. 長期運営
   - ドラフト、外国人補強、トレード、FA、契約更改、キャンプ育成を行う

## 6. 差別化ポイント

### 6.1 ライブペナント

現実のNPB日程と連動し、ユーザーのゲーム内試合も同じ相手・球場・先発予想で進む。現実の試合前に「自分ならこう組む」を試し、試合後に現実との差分を楽しめる。

### 6.2 采配介入は勝負所だけ

全打席操作ではなく、AIが重要局面を抽出する。

- 7回1点差、無死一二塁
- 先発が100球目前でクリーンナップ
- 左の代打が残っている場面
- 抑えが連投中の9回

これにより、1試合を3から5分で遊べる。

### 6.3 「納得できる」シミュレーション

結果だけでなく、なぜ勝った/負けたかを見せる。

- 期待得点
- 投手疲労
- 左右相性
- 守備範囲
- 球場補正
- 選手の好不調

野球ファンがSNSで語れる材料をゲーム内に作る。

### 6.4 推し球団モードとIFリーグ

- 推し球団モード: 現実シーズン連動。毎日触る主軸
- ドラゴンズ重点モード: 中日ドラゴンズを初期の看板体験にし、投手力、守備力、得点力改善、若手育成をゲームの主要課題として見せる
- IFリーグ: 「もし開幕から自分が監督なら」を通年で遊ぶ
- 歴代シナリオ: OBライセンスが取れる場合のみ拡張

### 6.5 課金が勝敗を壊しにくい設計

実在選手ガチャだけに寄せると既存大手と競合するため、課金価値を以下に分散する。

- シーズンパス: 報酬、分析レポート、限定演出
- スカウトレポート: 新外国人・ドラフト候補の詳細可視化
- 球団施設: 育成効率や疲労回復の長期強化
- 観戦連動パック: 現実の試合日に合わせた称号・ユニフォーム・演出
- プレミアムデータ: 詳細スタッツ、相手傾向分析、AIコーチ提案

## 7. ゲームシステム

### 7.1 選手パラメータ

- 打撃: ミート、長打、選球眼、対左右、バント、走塁
- 投球: 球威、制球、変化球、スタミナ、対左右、奪三振、ゴロ率
- 守備: 捕球、送球、肩、守備範囲、ポジション適性
- メンタル: 勝負強さ、連投耐性、若手成長、ベテラン安定感
- 状態: 疲労、好不調、怪我リスク、士気

### 7.2 試合エンジン

MVPでは簡易な確率モデルから開始する。

- 打者能力、投手能力、左右相性、球場、疲労、調子でイベント確率を算出
- 結果は三振、四球、単打、長打、本塁打、ゴロ、フライ、併殺、失策など
- 重要局面でプレイヤーに采配選択を提示
- 試合後に意思決定の影響をレポート化

将来的には実打球傾向、球種、配球、守備シフト、走者判断を追加する。

### 7.3 育成

- 二軍育成方針: 打撃改造、守備強化、球速アップ、変化球習得など
- 一軍起用育成: 若手を我慢して使うほど経験値が増えるが、短期勝率は下がる
- コーチ相性: コーチの得意分野で成長曲線が変化
- 怪我/疲労管理: 連戦時の休養判断が長期成績に影響

### 7.4 編成・補強

- ドラフト: スカウト地域、注目選手、隠し能力
- 外国人補強: 予算、適応リスク、日本球界相性
- トレード: チーム事情、年俸、将来性、ファン反応
- FA: 資金力だけでなく出場機会、地元、優勝可能性が影響
- 契約更改: 年俸総額と選手満足度を管理

### 7.5 実在データ連動

実在球団を使うほど、ゲームの分かりやすさと楽しさは強くなる。特に初期版では中日ドラゴンズを題材に、以下のデータをゲーム体験に変換する。

- 選手プロフィール: 年齢、投打、守備位置、背番号、在籍年数
- 直近成績: 打率、出塁率、長打率、防御率、WHIP、奪三振率など
- 起用傾向: 打順、守備位置、先発ローテ、勝ちパターン、代打/代走/守備固め
- コンディション風データ: 連投、出場間隔、直近打撃、二軍調整
- 球場特性: バンテリンドームを守備・投手・長打抑制の意思決定に反映
- チーム課題: 得点力、長打力、若手育成、リリーフ負荷などをシーズン目標にする

ただし、商用利用する場合は選手名、球団名、ロゴ、写真、ユニフォーム、詳細成績データの権利確認が必要になる。検証段階では、公開情報の手入力や架空化した近似データで「ドラゴンズを采配している感覚」を確認し、正式版ではライセンス取得済みデータへ差し替える。

## 8. MVP範囲

最初に作るべき最小構成は以下。

1. 中日ドラゴンズを最初の中心球団にする
2. ライセンス確認前は、正式名称・ロゴ・選手写真を使わず、検証用の近似/架空化データで作る
3. 対戦相手はセ・リーグ5球団相当のダミーチームを用意する
4. 30試合の短期ペナントを進行
5. 打順、先発、継投、代打、守備固めを設定
6. 試合は高速テキスト/2Dカード演出で進行
7. 重要局面で最大3回だけ采配介入
8. 試合後に勝因/敗因レポートを表示
9. 選手成長、疲労、簡易怪我、順位表を実装
10. デイリーミッションとシーズンパスの土台を実装

MVPの理想は「ドラゴンズファンが触って、これは自分の球団の悩みだと分かる」こと。全12球団対応は魅力的だが、最初はドラゴンズの解像度を高くし、球団固有の課題がゲーム上で表現できるかを優先する。

## 10. ライセンス・法務上の注意

実在プロ野球チーム・選手名・球団ロゴ・ユニフォーム・写真・成績データを商用利用するには、権利処理が重要になる。企画段階では以下を分けて進める。

1. ドラゴンズ検証版
   - 中日ドラゴンズを中心テーマにする
   - 非公開テストでは、手入力した公開情報や近似値を使い、ゲーム性を検証する
   - 外部公開前には、正式名称、選手名、ロゴ、ユニフォーム、写真、詳細成績データの利用可否を確認する
2. ライセンス交渉版
   - NPB、球団、選手会、データ提供会社、写真素材の権利を確認
   - 中日ドラゴンズを最優先の交渉対象にし、1球団特化の公式体験として成立するかを検討する
   - 契約範囲に応じて実名、ロゴ、選手写真、成績、ニュース連動の使用可否を切り分け
3. 代替案
   - ユーザー編集データ
   - 名古屋の球団を想起させすぎない架空チームで、投手力・守備力・得点力改善という戦略体験を残す
   - 将来的に公式ライセンス版へ移行

## 11. マネタイズ

### 11.1 推奨モデル

- 基本プレイ無料
- シーズンパス
- 月額プレミアム分析プラン
- 見た目アイテム
- 育成/スカウト時短
- 広告視聴による追加報酬

### 11.2 避けたいモデル

- 最強選手ガチャだけで勝敗が決まる
- 現実の推し球団を弱く感じさせる課金圧
- 対人ランキングで重課金が一方的に勝つ構造

本作は「勝てる選手を買う」より「判断材料を増やす」「育成の選択肢を広げる」「推し球団を飾る」に課金価値を置く。

## 12. 継続率を作る運用

- 毎朝: 今日の相手分析、AIコーチ提案
- 試合前: スタメン予想イベント
- 試合後: 現実の試合結果に絡めたチャレンジ
- 月曜: 週間ベスト采配
- 月末: 球団オーナー評価
- オールスター/交流戦/CS/日本シリーズ: 大型イベント
- ドラフト会議: 年間最大イベントの一つ

## 13. KPI仮説

- D1継続: 推し球団選択、初勝利、初采配成功で引き上げる
- D7継続: 週間順位、選手成長、現実試合連動で引き上げる
- D30継続: 月間シーズン、ドラフト候補、シーズンパスで引き上げる
- 課金率: 分析機能、パス報酬、推し球団装飾で引き上げる
- SNS拡散: 試合後レポート、采配評価、現実との差分投稿で引き上げる

## 14. リスクと対策

| リスク | 影響 | 対策 |
| --- | --- | --- |
| 実在チーム/選手ライセンス取得難度 | 企画の訴求が弱まる | 中日ドラゴンズを最優先に権利範囲を確認し、検証版では近似/架空化データに差し替えられる構造を残す |
| シミュレーションが理不尽に見える | 継続率低下 | 結果理由と采配影響を必ず可視化する |
| データ量が多く初心者が離脱 | 初回離脱 | AIおすすめ、簡単編成、重要局面だけ介入を用意 |
| 大手アクションゲームとの差別化不足 | 集客不利 | 3D操作ではなく、監督/GM/現実連動に集中 |
| 課金が勝敗に直結しすぎる | 炎上・短命化 | 課金は分析、装飾、時短、長期育成中心にする |

## 15. 初期プロトタイプの成功条件

1. 1試合が5分以内に終わる
2. 負けても「次はこうしたい」と思える
3. 試合後レポートがSNSに貼りたくなる
4. 推し選手を起用する理由がゲーム上も成立する
5. アクション操作なしでも野球をやっている感覚がある

## 16. 結論

勝ち筋は、プロスピAのようなリアルアクション大作に対抗することではなく、野球ファンの日常会話そのものをスマホゲームにすることにある。

「現実のチームを、自分の采配で少しだけ良くできる」という幻想は強い。特に中日ドラゴンズを最初の看板体験にすれば、投手力、守備、若手起用、得点力改善、バンテリンドームという分かりやすいテーマで、野球ファンがすぐに楽しさを理解できる。

Flutterではまず2D/テキスト中心で軽く作り、シミュレーションの納得感、試合後レポート、毎日触る導線を磨くべきである。実在ライセンスは最大の武器だが最大のリスクでもあるため、MVPではドラゴンズを強く意識した検証用データで中毒性を確認し、手応えが出た段階で公式ライセンス版へ進める。
